This report is a accounting of exploratory data analysis on three large corpus files containing blogs, news, and tweets from US English sources.The goal of this analysis is to get a sense of the complexity, size, and scope of the corpora by breaking them into words and phrases (n-grams) and graphing some simple relationships.

First, the files are downloaded and loaded into a working directory. The working directory is named and the files are referenced for easier retrieval

These files are then read into our environment so that they can be analyzed. (using a connection and the readLines function)

We will then create a document frequency matrix with each file (using the Quanteda package). This converts the words to lower case, removes numbers and punctuation, and calculates the frequencies of each word (or unigram).

With these initial files, we can then get a sense of the size and shape of the files. The Dim attribute of each DFM will give us the number of sentences and features(words) for each corpus.

# Blog Dimensions (documents and unigrams)
usb@Dim
## [1] 899288 387402
# News Dimensions (documents and unigrams)
usn@Dim
## [1] 1010242  336677
# Twitter Dimensions (documents and unigrams)
ust@Dim
## [1] 2360148  396632

We can see that the matrices have between 900k and 2.3MM sentences and between 330k and 440k words.

A histogram of each would show that the word distribution is overwhelmed by the few most common words like “the”, “of”, “is”, etc. These common words are called stop words but they have been left in our document frequency matrix because they will be invaluable in making word predictions.

A variable distribution plot shows the frequency of words by their rank. This helps to show graphically just how dominant the most common words are within each corpus. There is a long tail of hundreds of thousands of words that appear just a small handful of times, even though here we have just graphed the top 1000 features.

Although each corpus contains hundreds of thousands of words, 90% of the English language can be covered by just 2800 words (via the New General Service List). Moving forward, one of the biggest challenges will be trading off between accuracy and file size. The table below shows the number of unigrams, bigrams, and trigrams for each corpus. Any prediction model for the final project will need to appropriately resize these files to be able to work in a reasonable amount of time.

Set the working directory and name the files

setwd("~/Desktop/Coursera/Capstone")
library(quanteda)

blogs <- "./en_US/en_US.blogs.txt"
news <- "./en_US/en_US.news.txt"
twitter <- "./en_US/en_US.twitter.txt"
con <- file(blogs, "r")
USblogs <- readLines(con)
close(con)

con <- file(news, "r")
USnews <- readLines(con)
close(con)

con <- file(twitter, "r")
UStwitter <- readLines(con)
close(con)
usb <- dfm(USblogs, verbose = TRUE, toLower = TRUE, removeNumbers = TRUE, removeTwitter = TRUE, ngrams = 1)
usn <- dfm(USnews, verbose = TRUE, toLower = TRUE, removeNumbers = TRUE, removeTwitter = TRUE, ngrams = 1)
ust <- dfm(UStwitter, verbose = TRUE, toLower = TRUE, removeNumbers = TRUE, removeTwitter = TRUE, ngrams = 1)
# Blog Dimensions (documents and unigrams)
usb@Dim
# News Dimensions (documents and unigrams)
usn@Dim
# Twitter Dimensions (documents and unigrams)
ust@Dim
plot(topfeatures(usb, 1000), ylab = "Number of Times Word Appears", xlab = "Rank of Word", main = "Top Feature of Blog Corpus")
plot(topfeatures(usn, 1000), ylab = "Number of Times Word Appears", xlab = "Rank of Word", main = "Top Feature of News Corpus")
plot(topfeatures(ust, 1000), ylab = "Number of Times Word Appears", xlab = "Rank of Word", main = "Top Feature of Twitter Corpus")
